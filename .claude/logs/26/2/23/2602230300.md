# 별점(Importance) 필터 추가

## 사용된 프롬프트
- `.claude/prompts/26/2/23/0300.md`

## 작업 내용
뉴스와 상한가 기록 목록 조회 시 별점(importance score)으로 필터링하는 `minScore` 파라미터 추가

## 수정 파일

| 파일 | 변경 내용 |
|------|-----------|
| `application/provided/ImportanceRepository.kt` | `findTargetIdsByMemberIdAndTargetTypeAndMinScore` 쿼리 메서드 추가 |
| `application/required/NewsLister.kt` | `list()` 메서드에 `minScore: BigDecimal?` 파라미터 추가 |
| `application/impl/news/query/QueryNewsLister.kt` | ImportanceRepository 의존성 추가, minScore 필터 로직 구현, favoriteOnly와 교집합 처리 |
| `adapter/webapi/news/GetNewsListApi.kt` | `@RequestParam minScore` 추가, minScore 사용 시 memberId 추출 |
| `application/required/DailyTop30RecordLister.kt` | `listByDateRange()`, `listByPage()` 모두 `minScore` 파라미터 추가 |
| `application/impl/record/query/QueryDailyTop30RecordLister.kt` | ImportanceRepository 의존성 추가, `resolveFilteredIds()` 공통 메서드로 리팩토링 |
| `adapter/webapi/record/GetDailyTop30RecordListApi.kt` | `@RequestParam minScore` 추가, minScore 사용 시 memberId 추출 |

## 빌드 결과
- `compileKotlin` 성공
- 기존 테스트 오류(RedisTokenRepositoryTest)는 이번 변경과 무관
