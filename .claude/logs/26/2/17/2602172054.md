# Convention Rule 전체 적용 - Finder 인터페이스 사용 강제

## 사용된 파일
- .claude/rules/convention.md

## 작업 내용

convention.md 룰을 전체 도메인에 적용. 기존에는 member 도메인 예시만 적용되어 있었고, 나머지 도메인들은 repository를 직접 사용하고 있었음.

### 변경된 Finder 인터페이스
- **MemberFinder**: `findById` 반환타입 `Optional<Member>` → `Member` (내부에서 예외 처리)
- **NewsFinder**: `findEntityByIdWithPressAndThemes(id: Int): News` 추가
- **StockFinder**: `findEntityByIdWithThemes(id: Int): Stock`, `findAllEntitiesByIds(ids: List<Int>): List<Stock>` 추가
- **StockGroupFinder**: `findEntityByIdWithMember(id: Int): StockGroup`, `findEntityByIdWithStocks(id: Int): StockGroup` 추가
- **ThemeFinder**: `findAllEntitiesByIds(ids: List<Int>): List<Theme>` 추가
- **OpinionFinder**: 신규 생성 + `findEntityById(id: Int): Opinion`

### 신규 파일
- `application/required/OpinionFinder.kt`
- `application/impl/opinion/query/QueryOpinionFinder.kt`

### 수정된 Query 구현체
- QueryMemberFinder, QueryNewsFinder, QueryStockFinder, QueryStockGroupFinder, QueryThemeFinder

### 수정된 Command 클래스 (repository 직접 사용 → Finder 사용)
- CommandOpinionCreator: `.orElseThrow()` 제거 (MemberFinder가 내부 처리)
- CommandOpinionUpdater: `opinionRepository` → `opinionFinder`
- CommandStockGroupCreator: `.orElseThrow()` 제거
- CommandNewsThemeAdder: `newsRepository`, `themeRepository` → `newsFinder`, `themeFinder`
- CommandNewsThemeRemover: `newsRepository` → `newsFinder`
- CommandStockThemeAdder: `stockRepository`, `themeRepository` → `stockFinder`, `themeFinder`
- CommandStockThemeRemover: `stockRepository` → `stockFinder`
- CommandStockGroupDeleter: `stockGroupRepository.findByIdWithMember()` → `stockGroupFinder.findEntityByIdWithMember()`
- CommandStockGroupStockAdder: `stockGroupRepository`, `stockRepository` → `stockGroupFinder`, `stockFinder`
- CommandStockGroupStockRemover: `stockGroupRepository` → `stockGroupFinder`
- CommandStockGroupUpdater: `stockGroupRepository` → `stockGroupFinder`
