# Work Log - API 도메인별 패키징 분리

## 작업 내용
`1227.md` 프롬프트에 따라 `adapter/webapi` 하위의 API 컨트롤러와 DTO를 도메인별로 패키징 분리

## 변경 사항

### Before (flat structure)
```
adapter/webapi/
├── AuthApi.kt, GetMemberListApi.kt, ...  (31 controllers)
└── dto/
    ├── request/   (10 files)
    └── response/  (27 files)
```

### After (domain-based structure)
```
adapter/webapi/
├── auth/           - AuthApi, GetMemberListApi + 관련 DTOs
├── favorite/       - AddFavoriteApi, RemoveFavoriteApi, GetFavoriteListApi, CheckFavoriteApi + DTOs
├── stockgroup/     - Create/Update/Delete/List/Detail/AddStock/RemoveStock APIs + DTOs
├── theme/          - GetThemeList/Detail, AddNewsTheme, RemoveNewsTheme, AddStockTheme, RemoveStockTheme + DTOs
├── news/           - GetNewsList, GetNewsDetail + DTOs
├── opinion/        - CreateNews/RecordOpinion, UpdateNews/RecordOpinion, GetNews/RecordOpinionList + DTOs
├── stock/          - GetStockList, GetStockDetail + DTOs
├── record/         - GetDailyTop30RecordList + DTOs
└── financialstatement/ - GetFinancialStatementList + DTOs
```

### 수정된 파일 (약 70+ 파일)
- API 컨트롤러 31개: 패키지 선언 변경
- DTO 37개: 패키지 선언 변경 + cross-domain import 추가
- application/required 인터페이스들: import 경로 업데이트
- application/impl 구현체들: import 경로 업데이트
- AuthApiTest.kt: import 경로 업데이트

### Cross-domain DTO 참조 (import 추가)
- StockResponse: theme, record, financialstatement, stockgroup, opinion 도메인에서 참조
- ThemeResponse: stock, news, record, opinion 도메인에서 참조
- PressResponse: opinion 도메인에서 참조
- NewsResponse: theme 도메인에서 참조
