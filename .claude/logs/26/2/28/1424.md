# 첨부파일(Attachment) 단순화 리팩터링

## 작업일시
2026-02-28

## 요약
첨부파일을 다른 엔티티와 연관짓지 않고, 웹에디터에서 파일을 업로드/조회/다운로드/삭제하는 단순 구조로 변경

## 변경 내용

### 삭제한 파일
- `domain/attachment/AttachmentTargetType.kt` - enum 삭제

### 수정한 파일
1. `domain/attachment/Attachment.kt` - targetType, targetId 필드 제거
2. `application/provided/AttachmentRepository.kt` - findByTargetTypeAndTargetId → findByMemberId로 변경
3. `application/required/attachment/AttachmentUploader.kt` - targetType, targetId 파라미터 제거
4. `application/required/attachment/AttachmentLister.kt` - memberId로 조회하도록 변경
5. `application/impl/attachment/command/CommandAttachmentUploader.kt` - targetType, targetId 관련 코드 제거
6. `application/impl/attachment/query/QueryAttachmentLister.kt` - memberId로 조회
7. `application/impl/attachment/query/QueryAttachmentFinder.kt` - response에서 targetType, targetId 제거
8. `adapter/webapi/attachment/UploadAttachmentApi.kt` - targetType, targetId RequestParam 제거
9. `adapter/webapi/attachment/GetAttachmentListApi.kt` - memberId 기반 조회로 변경
10. `adapter/webapi/attachment/dto/response/AttachmentResponse.kt` - targetType, targetId 필드 제거

### 문서 업데이트
- `api_documents/attachment.md` - API 문서 업데이트

## DB 변경 (수동 실행 필요)
```sql
DROP INDEX IF EXISTS idx_attachments_target;
ALTER TABLE attachments DROP COLUMN IF EXISTS target_type;
ALTER TABLE attachments DROP COLUMN IF EXISTS target_id;
```
