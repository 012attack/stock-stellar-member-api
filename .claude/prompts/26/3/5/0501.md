stock_daily_prices 테이블 설명

데이터 소스

- pykrx 라이브러리 → KRX(한국거래소)에서 직접 일별 OHLCV 데이터를 가져옴
- API: stock.get_market_ohlcv(시작일, 종료일, 종목코드)

테이블 구조

├───────────────┼─────────────────────────┼──────────────────────────────┤
│ high_price    │ BIGINT                  │ 고가 (원)                    │
├───────────────┼─────────────────────────┼──────────────────────────────┤
│ low_price     │ BIGINT                  │ 저가 (원)                    │
├───────────────┼─────────────────────────┼──────────────────────────────┤

├───────────────┼─────────────────────────┼──────────────────────────────┤

제약 조건

- UNIQUE(stock_id, trade_date) — 같은 종목+같은 날짜 중복 불가
- INDEX(stock_id, trade_date) — 종목+날짜 조회 성능 최적화

수집 방식

- 증분 수집: 해당 종목의 MAX(trade_date) + 1일부터 오늘까지 수집
- 전체 수집: start_date 지정 시 해당 날짜부터 수집
- stocks 테이블에 없는 종목은 네이버 증권에서 종목명/기업개요를 가져와 자동 등록

API 설계 시 활용 예시

-- 특정 종목의 최근 시세 조회
SELECT * FROM stock_daily_prices
WHERE stock_id = ? ORDER BY trade_date DESC LIMIT 30;

-- 특정 기간 시세 조회
SELECT * FROM stock_daily_prices
WHERE stock_id = ? AND trade_date BETWEEN ? AND ?
ORDER BY trade_date;

-- 종목코드로 조회 (stocks 테이블 JOIN)
SELECT s.stock_code, s.stock_name, p.*
FROM stock_daily_prices p
JOIN stocks s ON s.id = p.stock_id
WHERE s.stock_code = '005930'
ORDER BY p.trade_date DESC;
